S-Cure CSV Import Guide

How to create and import cure programs via CSV files

1. Overview

The S-Cure system allows you to create custom cure programs by importing CSV (Comma-Separated Values) files. Each CSV file defines a sequence of process steps that the machine will execute in order. This guide explains the required format, valid parameters, and common errors to avoid.

1.1 Build Criteria — what makes a valid program

A program (whether built on-screen or imported from CSV) is accepted only when it meets all of these criteria:

A valid cure program must satisfy:
  • Has a name and at least one step.
  • Temperature only rises. Each Heating / Drying / Cure / Bleacher step must be at a temperature equal to or higher than the previous step. To use a lower temperature you must first add a Cooling step — a Cooling step resets the floor, after which the temperature may go lower again. (You cannot set a Heating step below the temperature already reached without cooling the chamber first.)
  • Temperature range 20–80°C (in 5° steps). A Cooling step's target is 20°C up to (previous step − 5) and must be below the previous step.
  • Time: 1–120 min for Drying / Heating / Cure, and 1–720 min (up to 12 h) for Bleacher. Cooling and Nitrogen carry no time.
  • UV Intensity 5–100% for Cure / Bleacher (default 30%).
  • Nitrogen order: at most 2 N₂ purges; every purge must be followed by Cure or Bleacher, then Cooling (N₂ → Cure/Bleacher → Cooling); two purges must be separated by a Cooling step.

On the on-screen builder these criteria are enforced live — the controls simply won't let you reach an invalid value, and Save is blocked with a message until the name / step / N₂ rules are met. On CSV import the same criteria are re-checked and a file that breaks any of them is rejected (see Section 7).

2. CSV File Structure

The CSV file must contain a header row followed by one or more data rows. Each row represents one process step. Columns are read by position in the order shown in Section 2.2 — keep them in that order. The header row itself is only checked for the word "Step" or "Process"; the actual column names are not parsed.

2.1 Header Format

Step,Process,Temperature,Time,TimerMode,UVIntensity,UVStart,UVRampPercent,CoolingMode

2.2 Column Reference

# Column Description Status
1 Step Step number (1, 2, 3...) Required
2 Process Process type (see Section 3) Required
3 Temperature Target temperature in °C (20–80) Optional*
4 Time Duration in minutes (1–120) Required**
5 TimerMode When the timer starts counting Cure/Bleacher
6 UVIntensity UV light intensity in % (0–100) Cure/Bleacher
7 UVStart When UV turns on Cure/Bleacher
8 UVRampPercent Ramp % at which UV turns on Optional
9 CoolingMode Cooling speed: fast, medium, or slow Cooling only

* Temperature applies to Drying, Heating, Cure, Bleacher, and Cooling (leave empty for Nitrogen). It is optional: if given it must be 20–80°C, otherwise the importer defaults it to 40°C (25°C for Cooling).
** Time is required for Drying, Heating, and Cure (1–120 min) and for Bleacher (1–720 min, up to 12 h). Leave empty for Cooling and Nitrogen.

3. Process Types

Process Description Required Fields Notes
Drying Removes moisture from the part using heat Temperature, Time —
Heating Heats the chamber to target temperature Temperature, Time —
Cure UV curing with 405nm LED light Time (1–120 min) Temperature optional (defaults 40°C). Supports TimerMode, UVIntensity, UVStart, UVRampPercent
Bleacher Bleaching with 450nm LED light Time (1–720 min) Long runs allowed (up to 12 h). Temperature optional (defaults 40°C). Supports TimerMode, UVIntensity, UVStart, UVRampPercent
Cooling Controlled cooling to target temperature — Temperature optional (defaults 25°C). CoolingMode: fast / medium / slow (default: medium). Time is ignored.
Nitrogen N₂ purge of the chamber — No temperature or time. Must be followed by Cure/Bleacher, then a Cooling step. Runs automatically only if N₂ is enabled; otherwise skipped.

4. Nitrogen (N₂) Purge Rules

Nitrogen purge steps have specific placement rules that must be followed:

N₂ Purge Requirements
  • Maximum 2 Nitrogen steps per program
  • Immediately after a Nitrogen step, only a Cure or Bleacher step is allowed (not Heating or Cooling)
  • Immediately after that Cure/Bleacher, only a Cooling step is allowed — the required order is Nitrogen → Cure/Bleacher → Cooling
  • A Cooling step is required after the Nitrogen purge to vent the nitrogen from the chamber
  • At least one Cure or Bleacher step must follow the N₂ purge
  • If N₂ is not enabled on the system, Nitrogen steps are skipped automatically during the cure process

4.1 Which step may follow which

Current step Allowed next step
Nitrogen (N₂ purge) Only Cure or Bleacher — not Heating, Drying, Cooling, or another Nitrogen
The Cure / Bleacher that directly follows an N₂ purge Only Cooling (to vent the nitrogen)
Heating, Drying, Cure, Bleacher, or Cooling (not part of an N₂ sequence) Any process — provided the temperature does not drop without a Cooling step, and Nitrogen only if a Cooling has occurred since the last purge and fewer than 2 purges have been used

In short, a nitrogen purge always forms the fixed block N₂ → Cure/Bleacher → Cooling. Anything else after an N₂ purge is rejected.

A Temperature or Time value outside its range rejects the whole file. For the optional Cure/Bleacher/Cooling fields (TimerMode, UVIntensity, UVStart, UVRampPercent, CoolingMode) an out-of-range or misspelled value is silently ignored and the default is used — it does not reject the file.

7. What Will Be Rejected

The system will not import a CSV file if any of the following errors are found:

Rejection Reasons (any one stops the whole file from importing)
  • No header or no data rows — the file needs a header row plus at least one data row
  • Missing header keyword — the header row must contain the word "Step" or "Process" (case-insensitive)
  • Less than 4 columns — each data row must have at least 4 comma-separated values
  • Invalid process type — must be exactly Drying, Heating, Cure, Bleacher, Cooling, or Nitrogen (case-sensitive)
  • Temperature out of range — if a temperature is given it must be 20–80°C
  • Time out of range — 1–120 min (Drying / Heating / Cure) or 1–720 min (Bleacher)
  • Temperature decreases without Cooling — a step's temperature cannot be lower than the previous step unless a Cooling step resets it
  • More than 2 Nitrogen steps — maximum 2 N₂ purge steps per program
  • Wrong step after Nitrogen — a Nitrogen step must be followed by Cure or Bleacher
  • Wrong step after the N₂ Cure/Bleacher — the step after the Cure/Bleacher that follows N₂ must be Cooling
  • Nitrogen not vented — a Cooling step is required after the N₂ purge
  • No Cure/Bleacher after Nitrogen — at least one Cure or Bleacher step must follow the N₂ purge
  • No valid steps — at least one valid data row is required
Not a rejection — silently defaulted

A misspelled or out-of-range TimerMode, UVIntensity, UVStart, UVRampPercent or CoolingMode does not reject the file — the importer ignores the bad value and applies the default (TimerMode on-target, UVStart at-target, UVIntensity 30, CoolingMode medium).

8. Timer Mode Explained

Mode Behavior
on-target (default) The step timer starts counting only after the chamber reaches the target temperature. The ramp-up time is not included in the step duration.
on-ramp The step timer starts counting immediately when the machine begins heating. The ramp-up time is included in the total step duration.

9. UV Start Mode Explained

Mode Behavior
at-start UV light turns on immediately when the step begins, even during ramp-up.
at-target (default) UV light turns on only after reaching the target temperature.

10. Cooling Mode Explained

Mode Behavior
fast Maximum fan speed for rapid cooling. Use when speed matters more than gradual temperature change.
medium (default) Balanced cooling speed. Recommended for most materials.
slow Gentle cooling for sensitive materials that may warp with rapid temperature change.

11. Quick Checklist

Before importing, verify:
  • File has .csv extension
  • First row is a header containing "Step" or "Process"
  • Each row has at least 4 columns, in the Section 2.2 order
  • Process types are spelled correctly (case-sensitive)
  • Any Temperature you provide is 20–80°C (omit it to accept the default: 40°C, or 25°C for Cooling)
  • Temperature and Time are left empty for Nitrogen steps
  • Time is left empty for Cooling steps
  • Time is 1–120 min (Drying / Heating / Cure) or 1–720 min (Bleacher)
  • Temperature never decreases between steps unless a Cooling step resets it
  • TimerMode, UVIntensity, UVStart and UVRampPercent are only meaningful for Cure or Bleacher
  • CoolingMode is only meaningful for Cooling steps
  • Maximum 2 Nitrogen steps per program
  • Order around N₂ is Nitrogen → Cure/Bleacher → Cooling
Important Notes

• Process names are case-sensitive: use "Heating" not "heating" or "HEATING".
• Empty optional fields should be left blank (not "0" or "null") — just leave nothing between the commas.
• The file must have a .csv extension.
• UTF-8 encoding is recommended. Avoid special characters in file names.
• If N₂ is not enabled on the system, Nitrogen steps are automatically skipped during cure.
